# Code of Conduct

This project is small on purpose: precise, readable, and useful.

## Expected behavior

- Be specific.
- Critique ideas, not people.
- Prefer concrete examples over vague arguments.
- Flag ambiguity with exact file names and section references.
- Treat interoperability, clarity, and user agency as first-order concerns.

## Unacceptable behavior

- Harassment, abuse, or hostility
- Deliberate spam or bad-faith proposals
- Misrepresenting the spec or examples
- Submitting AI-generated noise without review

## Enforcement

The steward may edit, close, or reject contributions that reduce clarity, trust, safety, or interoperability.
