# Security Policy

AIP is a document/specification repo, but security still matters.

## Report security concerns for

- prompt injection risks in packet examples
- ambiguous trust boundaries
- unsafe bridge automation assumptions
- parsing patterns that could encourage unsafe implementation

## How to report

Open a private channel if available, or create a minimal public issue that does not publish exploit details. Include:

- affected file
- exact section or example
- exploit path
- expected safe behavior
- recommended mitigation

## Scope

This repo does not ship a production service. Reports should focus on protocol-level and implementation-guidance risks.
